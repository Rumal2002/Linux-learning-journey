# Day 9 - Bash Scripting Essentials

## Overview

On Day 9 of my DevOps learning journey, I expanded my Bash scripting knowledge by exploring command substitution, environment variables, and command exit statuses.

These concepts are widely used in Linux automation, DevOps workflows, CI/CD pipelines, and cloud environments. Through practical exercises, I learned how to capture command outputs, access Linux environment variables, and verify whether commands execute successfully.

---

## Topics Covered

- Command Substitution (`$(...)`)
- Environment Variables
- Exit Status (`$?`)
- Working with Linux System Information
- Bash Automation Fundamentals

---

## Practical Scripts

| Script | Description |
|---------|-------------|
| my-profile.sh | Displays user information using variables and command substitution |
| environment.sh | Displays important Linux environment variables |
| check-command.sh | Demonstrates how to check the exit status of Linux commands |

---

## Key Learning Outcomes

- Learned how to capture command output using command substitution.
- Understood the purpose of Linux environment variables.
- Accessed built-in variables such as `$HOME`, `$USER`, `$PWD`, `$HOSTNAME`, and `$SHELL`.
- Learned how Linux reports command success and failure using exit status codes.
- Improved Bash scripting skills for automation and system administration.
