# Networking Notes

## What is a Computer Network?

A computer network is a collection of connected devices that communicate with each other to exchange data and share resources.

---

## Why Networking is Important for DevOps

Networking is a core skill for DevOps Engineers because modern applications run across multiple servers, cloud platforms, containers, and virtual networks.

It is essential for:

- Docker
- Kubernetes
- AWS
- CI/CD Pipelines
- Linux Server Administration

---

## Types of Networks

### PAN (Personal Area Network)

A small network used to connect personal devices such as smartphones, smartwatches, and Bluetooth devices.

Example:
- Bluetooth Earbuds
- Smart Watch

---

### LAN (Local Area Network)

A network that connects devices within a limited area such as a home, office, or university.

Example:
- Home Wi-Fi
- Office Network

---

### MAN (Metropolitan Area Network)

A network that spans across a city or metropolitan area.

Example:
- University campuses connected within the same city.

---

### WAN (Wide Area Network)

A network that connects devices over large geographical distances.

Example:
- The Internet

---

## Client vs Server

A Client sends requests.

A Server receives requests, processes them, and sends responses.

Example:

Web Browser → Web Server

---

## Common Network Devices

### Router

Connects different networks and forwards network traffic.

### Switch

Connects multiple devices within the same local network.

### Hub

Broadcasts data to all connected devices.

### Firewall

Monitors and filters incoming and outgoing network traffic.

### Modem

Connects a local network to an Internet Service Provider (ISP).

### Access Point

Provides wireless (Wi-Fi) connectivity to devices.

---

## Summary

Networking provides the foundation for cloud computing, Linux administration, containerization, and DevOps automation. Understanding these concepts is essential before learning Docker, Kubernetes, and cloud platforms.
